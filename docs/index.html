§ I

Three layers,
one repo.

Use as much or as little as you need. The MCP gives Claude capability. The plugin gives Claude judgment. The deployment gives the firm a place to host both.

Layer 01

MCP server

Forty-one typed tools across twelve Clio domains, plus a generic clio_api_request escape hatch. Same binary, two transports — stdio for local, stateless HTTP for Azure.

  • / OAuth 2.0 + AES-256-GCM at rest
  • / Append-only JSONL audit log
  • / US · CA · EU · AU regions
Layer 02

Claude Code plugin

Skills, agents, and an .mcp.json wiring that turns the MCP into ten domain workflows Claude can pick from automatically.

  • / Intake, time, billing, trust, docs
  • / Best-practice + setup walkthroughs
  • / Two agents: intake + read-only analyst
Layer 03

Azure deployment

One azd up provisions Container Apps, ACR, Key Vault, Files, App Insights. Secrets via managed identity — never on disk.

  • / Stateless HTTP, horizontal scale
  • / Timing-safe bearer auth
  • / ≈ $25–30/mo, 1 warm replica

§ VII

Designed for a law firm.

Tokens never touch chat. The audit trail holds up under ABA Op 512 scrutiny. Confidentiality is the default, not a setting.

Read the security doc →
Token at rest

AES-256-GCM

12-byte IV plus 16-byte GCM auth tag. Encrypted to $CLIO_STATE_DIR/tokens.json at mode 0o600.

Audit log

Append-only JSONL

Three modes: none, metadata, full. Per-firm retention configurable.

HTTP transport

Timing-safe bearer

Bearer tokens compared on SHA-256 digests via timingSafeEqual. caller-id is the first 6 bytes of the digest — auditable without storing tokens.

Azure

Managed identity

Secrets flow from Key Vault to the Container App via system-assigned MI. RBAC, soft-delete, purge protection on by default.

§ VIII

Six common questions.

01 Is this an official Clio product? +

No. It is an independent open-source integration that uses Clio's public v4 API. Clio does not endorse or support it. Bring your own developer app, your own OAuth credentials, and your own deployment.

02 Does my Clio data leave the firm? +

The MCP is a passthrough; it does not store Clio data. Calls go Claude → MCP → Clio API and back. The audit log records what was called (and optionally the args/results, per mode) and stays on whatever host runs the MCP. On Azure that's Azure Files inside your tenant.

03 Will it work with the Claude desktop app? +

Yes — the stdio transport works with any MCP client. An example claude_desktop_config.json lives in examples/. The Claude Code plugin layer is Claude Code specific.

04 How much does Azure cost? +

About $25–30 per month for one warm Container Apps replica plus Log Analytics, App Insights, Key Vault, and ACR Basic. Scale-to-zero is supported if you accept cold-start tradeoffs. ACR Basic is the dominant fixed cost.

05 What about ABA Formal Opinion 512? +

The MCP and plugin are designed around it. Per-call audit, explicit write confirmations, redaction-aware reads, and the clio-best-practices skill that loads Op 512 guidance into every relevant conversation. None of it replaces a lawyer's judgment — but it removes the most common ways AI accidentally violates the duties of competence and confidentiality.

06 Which Clio region am I in? +

Whichever subdomain you sign in at: app.clio.com (US), ca.app.clio.com (CA), eu.app.clio.com (EU), or au.app.clio.com (AU). Set CLIO_REGION to us · ca · eu · au. You must create the developer app on the matching regional portal.
