What you get

Three layers, one repository.

The MCP gives Claude capability. The plugin gives it judgment. The Azure deployment gives the firm a place to host both.

MCP server

Forty-one typed tools across twelve Clio domains. Same binary, two transports — stdio for local, stateless HTTP for Azure.

  • → OAuth 2.0 + AES-256-GCM at rest
  • → Append-only JSONL audit log
  • → US · CA · EU · AU regions

Claude Code plugin

Ten skills and two agents that wire the MCP into named workflows Claude picks from automatically.

  • → Intake, time, billing, trust, docs
  • → Best-practice + setup walkthroughs
  • → Intake + read-only analyst agents

Azure deployment

One azd up provisions Container Apps, ACR, Key Vault, Files, App Insights — secrets via managed identity.

  • → Stateless HTTP, horizontal scale
  • → Timing-safe bearer auth
  • → ≈ $25–30/mo, 1 warm replica
Install

Five minutes to your first call.

The local path is three terminal commands and one OAuth dance. For firm-wide deployment, swap step three for azd up.

1

Clone & build

git clone https://github.com/patrickking67/clio-manage-mcp
cd clio-manage-mcp
npm install
npm run build
2

Configure OAuth

cp .env.example .env
# Create app at developers.clio.com
# Redirect URI:
#   http://localhost:53682/oauth/callback
# Paste CLIO_CLIENT_ID + SECRET
3

Run & verify

npm run start:stdio
# in another shell:
npm run smoke:stdio
# → OK 41 tools

Add to Claude Code

~/.claude/mcp_servers.json 
{
  "mcpServers": {
    "clio-manage": {
      "command": "node",
      "args": ["/path/to/clio-manage-mcp/build/index.js", "--stdio"],
      "env": { "CLIO_REGION": "us" }
    }
  }
}

Then claude /mcp lists clio-manage ✓ with 41 tools.

Security

Designed for a law firm.

Tokens never touch chat. The audit trail holds up under ABA Op 512 scrutiny. Confidentiality is the default — not a setting.

Read the security doc
Token at rest

AES-256-GCM

12-byte IV plus 16-byte auth tag. Encrypted to tokens.json at mode 0o600.

Audit log

Append-only JSONL

Three modes — none, metadata, full.

HTTP transport

Timing-safe bearer

SHA-256 digests via timingSafeEqual. Caller-ID is the first 6 bytes.

Azure

Managed identity

Secrets flow from Key Vault to the Container App via MI. RBAC + soft-delete on by default.

FAQ

Six common questions.

Is this an official Clio product? +

No. It's an independent open-source integration that uses Clio's public v4 API. Clio doesn't endorse or support it. Bring your own developer app, your own OAuth credentials, and your own deployment.

Does my Clio data leave the firm? +

The MCP is a passthrough; it doesn't store Clio data. Calls go Claude → MCP → Clio API and back. The audit log records calls (and optionally args/results per mode) and stays on whatever host runs the MCP. On Azure that's Azure Files inside your tenant.

Will it work with the Claude desktop app? +

Yes — the stdio transport works with any MCP client. An example claude_desktop_config.json ships in examples/. The plugin layer is Claude Code specific.

How much does Azure cost? +

About $25–30/month for one warm Container Apps replica plus Log Analytics, App Insights, Key Vault, and ACR Basic. Scale-to-zero is supported if cold-start tradeoffs are acceptable.

What about ABA Formal Opinion 512? +

Designed around it. Per-call audit, explicit write confirmations, redaction-aware reads, and the clio-best-practices skill that loads Op 512 into every relevant conversation. None of that replaces a lawyer's judgment — but it removes the most common ways AI accidentally trips the duties of competence and confidentiality.

Which Clio region am I in? +

Whichever subdomain you sign in at — app.clio.com (US), ca.app.clio.com (CA), eu.app.clio.com (EU), or au.app.clio.com (AU). Set CLIO_REGION to match. Build the developer app on the matching regional portal.
