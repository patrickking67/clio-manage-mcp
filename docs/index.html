What's in the box

Three layers, all in one repo. Use as much or as little as you need.

MCP server

41 typed tools across 12 Clio domains, plus a generic clio_api_request escape hatch. Same binary, two transports — stdio for local, stateless HTTP for Azure.

  • › OAuth 2.0 + AES-256-GCM token-at-rest
  • › Append-only JSONL audit log (ABA Op 512)
  • › US / CA / EU / AU regions

Claude Code plugin

Skills, agents, and an .mcp.json wiring that turns the MCP into ten domain workflows Claude can pick from automatically.

  • › Intake, time, billing, trust, docs, calendar
  • › Best-practice + setup walkthroughs
  • › Two agents: intake + read-only analyst

Azure deployment

One azd up provisions Container Apps, ACR, Key Vault, Azure Files, Log Analytics + App Insights. Secrets flow via managed identity — never on disk.

  • › Stateless HTTP transport, horizontal scale
  • › Timing-safe bearer auth, SHA-256 token digests
  • › ~$25–30/month for one warm replica

Install

The local stdio path takes about five minutes. Azure takes one azd up.

1

Clone & build

git clone https://github.com/patrickking67/clio-manage-mcp
cd clio-manage-mcp
npm install
npm run build
2

Configure OAuth

cp .env.example .env
# Create app at developers.clio.com
# Redirect URI:
#   http://localhost:53682/oauth/callback
# Paste CLIO_CLIENT_ID + CLIO_CLIENT_SECRET
Full OAuth walkthrough →
3

Run & smoke test

npm run start:stdio
# In another shell:
npm run smoke:stdio
# → OK 41 tools

Wire into Claude Code

Add to ~/.claude/mcp_servers.json (or project-scoped .mcp.json): 

{
  "mcpServers": {
    "clio-manage": {
      "command": "node",
      "args": ["/absolute/path/to/clio-manage-mcp/build/index.js", "--stdio"],
      "env": { "CLIO_REGION": "us" }
    }
  }
}

Then claude /mcp should list clio-manage ✓ with 41 tools.

Security & compliance

Designed for production at a law firm. Encryption at rest, OAuth tokens that never touch chat, and an audit trail that holds up under ABA Formal Opinion 512 scrutiny.

Read the security doc →

Token-at-rest

AES-256-GCM with 12-byte IV + 16-byte GCM auth tag. Encrypted in $CLIO_STATE_DIR/tokens.json at mode 0o600.

Audit log

Append-only JSONL. Three modes: none, metadata, full. Per-firm retention controllable.

HTTP transport

Bearer tokens compared with timing-safe equality on SHA-256 digests. CallerID = first 6 bytes of digest — auditable without storing the token.

Azure managed identity

Secrets flow from Key Vault to the Container App via system-assigned MI. RBAC, soft-delete, purge protection on by default.

FAQ

Is this an official Clio product? +

No. It's an independent open-source integration that uses Clio's public v4 API. Clio doesn't endorse or support it. Bring your own developer app, your own OAuth credentials, and your own deployment.

Does my Clio data leave the firm? +

The MCP is a passthrough — it doesn't store Clio data. Tool calls go from Claude → MCP → Clio API → back. The audit log records what was called (and optionally the args/results, depending on mode), and that file stays on whatever host runs the MCP. If you deploy to Azure, the audit log lives in Azure Files inside your tenant.

Can I use this with the Claude desktop app? +

Yes, the stdio transport works with any MCP client. Example claude_desktop_config.json is in the examples folder. The Claude Code plugin layer is Claude Code-specific.

How much does it cost to run on Azure? +

About $25–30/month for a single warm Container Apps replica + Log Analytics + App Insights + Key Vault + ACR. Scale-to-zero is supported if you want cold-start tradeoffs. ACR Basic is the dominant fixed cost.

What about ABA Formal Opinion 512? +

The MCP and plugin are designed around it. Per-call audit, explicit write confirmations, redaction-aware reads, and the clio-best-practices skill that loads ABA Op 512 guidance into every relevant conversation. None of that replaces a lawyer's judgment, but it removes the most common ways AI accidentally violates the duties of competence and confidentiality.

Which Clio region am I in? +

Whichever subdomain you sign in at: app.clio.com (US), ca.app.clio.com (CA), eu.app.clio.com (EU), or au.app.clio.com (AU). Set CLIO_REGION to one of us / ca / eu / au. You must create the developer app on the matching regional portal.
